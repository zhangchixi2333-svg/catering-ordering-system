# 普通用户订单和支付功能使用指南

## 📋 概述

为普通用户（USER角色）添加了两个新功能页面：
1. **我的订单** - 查看和管理餐饮订单
2. **支付订单** - 查看和管理支付记录

---

## ✅ 已完成的功能

### 1. 新增页面

#### 📦 我的订单 (MyOrdersView.vue)
**路径**: `/my-orders`  
**功能**:
- ✅ 查看当前用户的所有订单
- ✅ 按订单状态筛选（待接单/制作中/已完成/已取消）
- ✅ 显示订单详细信息（订单号、桌台号、金额、支付方式等）
- ✅ 取消订单（仅待接单状态）
- ✅ 去支付（已完成但未支付的订单）
- ✅ 查看订单详情

#### 💳 支付订单 (PaymentOrdersView.vue)
**路径**: `/payment-orders`  
**功能**:
- ✅ 查看当前用户的所有支付订单
- ✅ 按支付状态筛选（待支付/支付中/支付成功/支付失败/已退款）
- ✅ 显示支付详细信息（支付单号、关联订单、金额、交易流水号等）
- ✅ 模拟支付成功（沙盒测试功能）
- ✅ 查看支付订单详情

---

### 2. 路由配置

在 `router/index.js` 中添加了两个新路由：

```javascript
{
  path: 'my-orders',
  name: 'MyOrders',
  component: () => import('../views/MyOrdersView.vue')
},
{
  path: 'payment-orders',
  name: 'PaymentOrders',
  component: () => import('../views/PaymentOrdersView.vue')
}
```

## 🚀 使用方法

### 1. 重启前端服务

修改完成后，需要重启前端开发服务器：

```bash
cd frontend
npm run dev
```

---

### 2. 访问页面

以普通用户身份登录后，左侧菜单会显示：

```
🏠 首页
🍽️ 在线点餐
🎫 取号排队
📦 我的订单      ← 点击这里查看订单
💳 支付订单      ← 点击这里查看支付记录
```

---

### 3. 我的订单页面功能

#### 查看订单列表
- 自动加载当前用户的所有订单
- 按创建时间倒序排列（最新的在前）

#### 筛选订单
- 选择"订单状态"下拉框
- 可选择：全部/待接单/制作中/已完成/已取消

#### 取消订单
1. 找到状态为"待接单"的订单
2. 点击"取消订单"按钮
3. 确认取消操作

#### 去支付
1. 找到状态为"已完成"但"未支付"的订单
2. 点击"去支付"按钮
3. 系统自动创建支付订单
4. 跳转到"支付订单"页面

#### 查看详情
- 点击"查看详情"按钮
- 弹窗显示订单完整信息

---

### 4. 支付订单页面功能

#### 查看支付列表
- 自动加载当前用户的所有支付订单
- 按创建时间倒序排列

#### 筛选支付
- 选择"支付状态"下拉框
- 可选择：全部/待支付/支付中/支付成功/支付失败/已退款

#### 模拟支付成功（沙盒测试）
1. 找到状态为"支付中"的支付订单
2. 点击"✅ 模拟支付成功"按钮
3. 确认操作
4. 系统调用沙盒接口模拟支付
5. 支付状态变为"支付成功"
6. 关联订单状态变为"待接单"

#### 查看详情
- 点击"查看详情"按钮
- 弹窗显示支付订单完整信息

## 🎯 完整的业务流程示例

### 场景1：用户下单并支付

```
1. 用户在"在线点餐"页面点餐
   ↓
2. 提交订单（订单状态：待接单）
   ↓
3. 进入"我的订单"页面
   ↓
4. 找到刚创建的订单
   ↓
5. 点击"去支付"按钮
   ↓
6. 系统创建支付订单（支付状态：支付中）
   ↓
7. 自动跳转到"支付订单"页面
   ↓
8. 点击"✅ 模拟支付成功"（沙盒测试）
   ↓
9. 支付成功！订单状态变为"待接单"
```

---

### 场景2：查看历史订单

```
1. 进入"我的订单"页面
   ↓
2. 查看所有历史订单
   ↓
3. 使用筛选功能查找特定状态的订单
   ↓
4. 点击"查看详情"查看完整信息
```

## 🔧 技术实现细节

### API 调用

#### 我的订单
```javascript
// 获取用户订单
const result = await orderApi.getByUser(userId)

// 取消订单
const result = await orderApi.cancel(orderId, reason)

// 创建支付订单
const result = await paymentApi.create({ orderNo })
```

#### 支付订单
```javascript
// 获取用户支付订单
const result = await paymentApi.getByUser(userId)

// 模拟支付成功（沙盒）
const response = await fetch(`/api/payment/sandbox/success/${id}`, {
  method: 'POST'
})
```

## ⚠️ 注意事项

### 1. 必须重启前端服务
修改了路由和菜单配置后，必须重启前端服务才能生效。

### 2. 后端服务要求
确保以下服务已启动：
- ✅ eureka-server (端口 8761)
- ✅ order-service (端口 8083)
- ✅ payment-service (端口 8084)
- ✅ gateway-service (端口 8087) - 可选

### 3. 沙盒测试
"模拟支付成功"功能仅在开发和测试环境使用，生产环境应禁用。

### 4. 权限控制
这两个页面仅对登录用户可见，未登录会自动跳转到登录页。
