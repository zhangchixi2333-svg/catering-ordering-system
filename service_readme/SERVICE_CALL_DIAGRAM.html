🏪 Shop Service

端口: 8081
店铺管理服务，提供店铺信息、桌台管理等基础数据
  • 店铺CRUD操作
  • 桌台管理
  • 营业状态管理
  • 被其他服务调用验证
基础服务 被动调用

🍜 Menu Service

端口: 8082
菜品管理服务，管理分类、菜品、库存等
  • 分类管理
  • 菜品CRUD
  • 库存管理
  • 调用Shop验证店铺
FeignClient Shop验证

📋 Order Service

端口: 8083
订单管理服务，处理订单创建、状态流转等
  • 订单创建
  • 订单状态管理
  • 调用Shop验证店铺
  • 调用Queue验证排队
FeignClient×2 核心业务

🎫 Queue Service

端口: 8085
排队管理服务，处理取号、叫号等
  • 在线取号
  • 叫号管理
  • WebSocket实时通知
  • 调用Shop验证+营业状态
FeignClient 营业检查

💰 Payment Service

端口: 8084
支付管理服务，处理支付订单、支付回调等
  • 支付订单创建
  • 支付回调处理
  • 调用Order验证订单
  • 防止重复支付
FeignClient Order验证

📢 Notification Service

端口: 8086
通知服务，多渠道消息推送
  • 短信通知
  • 邮件通知
  • WebSocket推送
  • RabbitMQ异步处理
基础设施 被动调用

🔄 典型调用流程

场景1: 用户取号就餐

Queue Service → Shop Service (验证店铺+营业状态) → 生成排队号码 → WebSocket推送通知

场景2: 用户点餐下单

Order Service → Shop Service (验证店铺) + Queue Service (验证排队) → 创建订单 → Notification Service (发送通知)

场景3: 用户支付订单

Payment Service → Order Service (验证订单+防重复支付) → 创建支付订单 → 第三方支付 → 支付回调 → Notification Service (发送通知)

场景4: 商家管理菜品

Menu Service → Shop Service (验证店铺) → 创建/更新菜品 → 更新库存
