```
GET /api/order/user/{userId}
```

```json
{
  "code": 200,

  "data": [
    {
      "id": 1,
      "orderNo": "ORD2026052000001",
      "tableNumber": "A01",
      "orderStatus": 0,
      "paymentStatus": 0,
      "totalAmount": 98.00,
      "actualAmount": 98.00,
      "paymentMethod": 1,
      "createTime": "2026-05-20T14:30:00"
    }
  ]
}
```

**验证点**:
- [ ] 请求 URL 正确
- [ ] 返回状态码 200
- [ ] data 是数组
- [ ] 订单字段完整

### 测试 2: 取消订单

**请求**:
```
PUT /api/order/1/cancel?cancelReason=用户主动取消
```

**预期响应**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": true
}
```

**验证点**:
- [ ] 请求方法：PUT
- [ ] 路径参数：订单ID
- [ ] 查询参数：cancelReason=用户主动取消
- [ ] 返回 data: true

**常见错误**:
- ❌ 404 Not Found: 订单不存在
- ❌ 400 Bad Request: cancelReason 参数缺失
- ❌ 500 Internal Error: 后端服务异常

```json
  "data": true
}
```

**验证点**:
- [ ] 请求方法：POST
- [ ] Content-Type: application/json
- [ ] 请求体包含 orderNo
- [ ] 返回 data: true

## 🐛 常见问题排查

### 问题 1: 页面样式不正确

**症状**: 
- 卡片没有圆角
- 按钮颜色不对
- 状态徽章不是胶囊形状

**排查步骤**:
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新页面（Ctrl+F5）
3. 检查浏览器控制台是否有 CSS 错误
4. 检查 Vite 是否正常编译

**解决方案**:
```powershell
# 重启前端开发服务器
cd frontend
npm run dev
```

### 问题 2: 取消订单失败

**症状**: 
- 点击"取消订单"后提示失败
- Network 标签中看到 400 或 500 错误

**排查步骤**:
1. 打开浏览器控制台查看错误信息
2. 检查 Network 标签中的请求详情
3. 确认请求 URL 和参数
4. 检查后端日志

**可能原因**:
- ❌ 后端 Order Service 未启动
- ❌ cancelReason 参数未传递
- ❌ 订单状态不允许取消（只有待接单可取消）
- ❌ 订单ID不正确

**解决方案**:
```powershell
# 检查后端服务状态
# 访问 Eureka 查看服务注册情况
http://localhost:8761

# 查看 Order Service 日志
# 确认服务正常运行
```

### 问题 3: 按钮点击无反应

**症状**: 
- 点击任何按钮都没有反应
- 控制台没有错误信息

**排查步骤**:
1. 检查浏览器控制台是否有 JavaScript 错误
2. 检查 Vue 组件是否正确挂载
3. 检查事件绑定是否正确

**解决方案**:
1. 刷新页面
2. 检查路由配置
3. 重新编译前端代码

### 问题 4: 订单列表为空

**症状**: 
- 登录后显示"暂无订单"
- 但数据库中应该有订单

**排查步骤**:
1. 检查 userStore 中是否有用户ID
2. 检查 API 调用是否成功
3. 检查后端返回的数据

**解决方案**:
```javascript

console.log('用户ID:', useUserStore().user?.id)
```
